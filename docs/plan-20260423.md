# wcmux 实施计划

基于 `docs/spec-20260423.md` 与 `docs/testing-20260423.md`，分 9 个阶段交付。

**平台目标**: Linux / macOS / Windows 三端均支持。PTY 与 cwd 读取按平台分派，上层接口统一。

---

## 0. 技术选型

| 领域 | 选择 | 理由 |
|---|---|---|
| Web 框架 | **FastAPI** + uvicorn | spec 明示；原生 WebSocket |
| 会话 | **`starlette.middleware.SessionMiddleware`** + itsdangerous | 无需外部存储，满足 §3.2 |
| 模板 | **Jinja2** + 服务端注入 `base_url` | 前端资源路径随部署路径自适应 |
| PTY (Unix) | **`ptyprocess`** | 稳定的纯 Python PTY 封装 |
| PTY (Windows) | **`pywinpty`**（基于 ConPTY，Win10 1809+）| Windows 唯一成熟方案 |
| cwd 读取 | **`psutil.Process(pid).cwd()`** | 三平台统一 API，内部各走 `/proc` / `proc_pidinfo` / `NtQueryInformationProcess` |
| 前端终端 | **xterm.js** + `xterm-addon-fit`（CDN）| spec §4.4 指定 |
| 前端框架 | **vanilla JS** + 单 HTML | Tab 状态机简单，无需框架 |
| 打包 | **`pyproject.toml` + `[project.scripts]`** | `pip install -e .` 后得到 `wcmux` 命令 |

**跨平台抽象**：新增 `terminal.py` 中 `Terminal` 接口，`terminal_unix.py` / `terminal_windows.py` 两份实现，`__init__` 时按 `sys.platform` 分派。对上层 `Tab` 逻辑完全透明。

## 3. 单 Tab 终端打通 — §4.1 子集 + §4.4

**跨平台 Terminal 接口**（`terminal.py`）
```python
class Terminal(Protocol):
    pid: int
    def write(self, data: str) -> None: ...
    async def read(self) -> str: ...      # 返回一次可读的输出片段
    def resize(self, rows: int, cols: int) -> None: ...
    def close(self) -> None: ...           # Unix: SIGHUP; Win: close handles
    @property
    def alive(self) -> bool: ...
```

**Unix 实现**（`terminal_unix.py`）
- `ptyprocess.PtyProcessUnicode.spawn([shell])`
- 用 `loop.add_reader(fd, callback)` 非阻塞读，回调里把数据塞进 `asyncio.Queue`
- `close()` 发 SIGHUP + `waitpid(WNOHANG)` 兜底

**Windows 实现**（`terminal_windows.py`）
- `winpty.PtyProcess.spawn([shell])`（`pywinpty` 包）
- `pywinpty` 提供 `read()` 非阻塞与 overlapped I/O；用 `loop.run_in_executor` 跑一个读循环，输出转发到 `asyncio.Queue`，避免 `add_reader` 在 Windows 上的限制
- `close()` 调用 `pty.close()`，内部会终止子进程

**WebSocket 协议**（文本 JSON）
- 上行：`{"type":"input","data":"..."}` / `{"type":"resize","rows":N,"cols":N}`
- 下行：`{"type":"output","data":"..."}` / `{"type":"exit","code":N}` / `{"type":"cwd","display":"...","full":"..."}` / `{"type":"replay","data":"..."}`

**前端**：`terminal.html` 挂载 xterm.js，先写死一个 tab 跑通 echo / vim / PowerShell。

**交付标志**：三平台各自单 Tab 能跑 vim(Unix) 或 pwsh(Win)、中文正常、尺寸随浏览器窗口变化。

## 4. 多 Tab 前后端 — §4.1 §4.2 §4.3

**后端**（`sessions.py`）
```python
@dataclass
class TabState:
    tab_id: str
    name: str
    terminal: Terminal
    buffer: collections.deque[str]   # 最近 ~256KB 输出，回放用
    cwd_full: str = ""
    cwd_display: str = ""
    active_ws: int = 0
    pending_close_task: Optional[asyncio.Task] = None

class UserSession:
    tabs: OrderedDict[str, TabState]  # 保持顺序
    MAX_TABS = 20

class SessionRegistry:
    sessions: dict[str, UserSession]   # key = session sid
```

**REST**
- `POST /api/tabs` → 创建并返回 `tab_id` + 初始 name；超 20 返 409
- `DELETE /api/tabs/{id}` → 关闭 + 清理
- `PATCH /api/tabs/{id}` body `{"name": "..."}` → 重命名
- `GET /api/tabs` → 当前 session 全部 Tab（刷新恢复用）

**前端**（`app.js`）
- `state = { tabs: [...], activeId }`，每 Tab 独立 xterm 实例 + WS
- 切换 = 隐藏/显示 DOM（保留 xterm scrollback），不销毁
- 未读标记：未激活 Tab 收到 `output` → `.unread`；激活清除
- 双击 Tab 名触发 `prompt` → `PATCH /api/tabs/{id}`

**交付标志**：testing.md §4.1 §4.2 §4.3 全部通过。

## 5. 刷新重连 & 5 分钟保留 — §4.5

- `TabState.buffer` 存最近输出环形队列（256KB 上限）
- 新 WS 连上时立即发 `{"type":"replay","data":buffer_concat}`，之后正常推送
- `active_ws==0` 时 `asyncio.create_task(wait_and_cleanup(tab, 300))`；重连时 `cancel()`
- 登出：遍历 session 所有 Tab `terminal.close()`，立即清理
- 前端启动顺序：先 `GET /api/tabs` → 有则恢复 UI → 每 Tab 连 WS → 收到 `replay` 写入 xterm

**交付标志**：关 2 分钟回得来、关 > 5 分钟是空白、登出立即终止进程。

## 6. Tab 名字 + cwd 显示 — §4.6

**跨平台 cwd**（`cwd.py`）
```python
import psutil
def read_cwd(pid: int) -> Optional[str]:
    try:
        return psutil.Process(pid).cwd()
    except (psutil.NoSuchProcess, psutil.AccessDenied):
        return None
```
`psutil` 内部在 Linux 读 `/proc/<pid>/cwd`、macOS 用 `proc_pidinfo`、Windows 用 `NtQueryInformationProcess`，对上层统一。

**轮询任务**：`SessionRegistry.start_cwd_poller()` 启单协程，每 **2 秒**遍历所有活跃 Tab
- 新 cwd 与旧一致 → 不推送（满足 testing §4.6 "无变化时不产生多余 WS 推送"）
- 变化 → 计算简写（在后端）：
  - 用户 `$HOME` 从环境（启动时捕获）取，替换前缀为 `~`
  - 长度 > 40：保留前 `N` + `…` + 末两段（如 `/a/b/.../y/z`）
  - 推送 `{"type":"cwd","display":"...","full":"..."}`

**前端**：Tab DOM 渲染 `<span class="name">${name}</span><span class="sep"> — </span><span class="cwd" title="${full}">${display}</span>`；进程死亡（`cwd_full=""`）时隐藏 `.sep` 与 `.cwd`。

**交付标志**：testing.md §4.6 全部通过（含跨平台 cd 验证）。

## 跨平台风险与应对

### R1. Windows ConPTY 要求 Win10 1809+
`pywinpty` 最新版依赖 ConPTY，旧 Win 退回 winpty legacy 方案。**处置**：README 写明"Windows 10 1809 及以上"作为支持下限，更旧版本不在支持范围。

### R2. Windows 下 asyncio 事件循环差异
`loop.add_reader` 在 `ProactorEventLoop` 上不可用。**处置**：Windows PTY 实现用 `loop.run_in_executor` 跑阻塞读循环，把数据通过 `asyncio.Queue`（线程安全包装 `call_soon_threadsafe`）送回主 loop。Unix 继续用 `add_reader`，两边在 `Terminal` 抽象之下对上层无差别。

### R3. Windows 默认 shell 行为差异
`cmd.exe` 对 ANSI / UTF-8 支持有限。**处置**：默认 shell 选择优先 `powershell.exe`（`%COMSPEC%` 通常是 `cmd.exe`，但 PowerShell 对 ANSI 友好度更高），README 注明并允许用户通过 `--shell` 覆盖。**需要 spec §4.1 补充 `--shell` 配置项**。

### R4. cwd 读取在 macOS 上需要权限
`psutil.Process.cwd()` 在 macOS 上偶发 `AccessDenied`（尤其在 SIP 限制下读不到子进程 cwd）。**处置**：`read_cwd` 捕获 `AccessDenied` 返回 `None`，Tab 标签退化到只显示名字，不让整条链路崩。

### R5. cwd 只反映 shell 本身，不反映前台子进程
spec §4.6 非目标已声明。跨平台下此行为一致（Linux/macOS/Windows 的 cwd API 都看 shell 进程自身）。

### R6. shell 语义差异影响用户心智
Unix 下 `cd` / `pwd` 基础命令跨 bash/zsh/fish 一致；Windows PowerShell 里 `cd` 也映射到 `Set-Location`，`pwd` 也有别名，**cwd 追踪跨平台无差异**。但 Tab 中的 shell 命令（ls vs dir、which vs where）对用户是暴露的——这是非目标，不处理。
